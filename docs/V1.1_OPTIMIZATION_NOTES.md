# DataCenter PowerInsight V1.1 优化说明

## 本次优化目标

把当前 App 从单文件 Demo 调整为更适合后续上线、接入 AI Agent 和知识库的项目结构。

## 已完成

1. 将原本写在 `src/App.jsx` 中的原型数据拆分到：
   - `src/data/marketData.js`
2. 将筛选、评分、摘要生成等业务逻辑拆分到：
   - `src/utils/insightUtils.js`
3. 保留现有 UI 和交互，不改变页面视觉和使用方式。
4. 已完成本地构建验证：`npm run build` 通过。

## 当前结构建议

```text
src/
├── App.jsx                # 页面组合与主交互
├── App.css                # 全局样式
├── main.jsx               # React 入口
├── data/
│   └── marketData.js      # 原型数据与数据源登记表
└── utils/
    └── insightUtils.js    # 筛选、评分、摘要生成逻辑
```

## 下一步建议

### V1.2

新增 `Ask PowerInsight` 页面，但先不直接暴露 API Key。

### V1.3

接入后端 API 或 Dify Workflow，由后端代理 Gemini / DeepSeek / Qwen 调用。

### V1.4

增加正式知识库：公司、产品、技术、新闻、报告五类数据源。

## 上线前检查

- 不要在前端代码中写入 Gemini API Key。
- 所有真实 API Key 应放在 Vercel / 后端环境变量中。
- 当前数据仍是专家整理原型数据，不应作为正式市场数据库对外销售。
